Semantic HTML

Describing what content is rather than how it looks.

Tag Name Description
<header> Header Within the body, this tag ids the title
<nav> Nav Within the header, this ids the navigation links
<main> Main The rest of the content (except for the footer) goes in here
<footer> Footer Right now this is not a terribly useful section. Its where I'd put contact info, copyright, terms of use, site map, or more nav links
<section> Section This wraps around content with the same theme
<article> Article This wraps around content that can make sense on its own
<aside> Aside Put content here that is extra. This is also where ads go
<figure> Figure This wraps around all media
<figcaption> Caption Use this to make a comment about the photo that users and screen readers both see
<video> Video In addition to src, video takes controls (play/pause), loop (loops through video), and autoplay. The text you put between <video></video> is what screen readers get or people when the video does not load
<audio> Audio Like video, this takes src and controls, though src goes into a <source> tag. Audio also takes type to specify the audio format like audio/mp3
<embed> Embed This brings in media from external sources. This is depreciated and you should use video, audio and img instead
<link> Link This is not actually semantic html, but I find it more descriptive than <a>
